# ระบบตรวจสอบสลิปโอนเงินอัตโนมัติผ่าน LINE OA (Automated Bank Slip Verification via LINE OA)

## 📌 แนวคิดและปัญหาที่พบ (Concept & Pain Points)
เปลี่ยนการเก็บข้อมูลสลิปแบบเดิมๆ ที่ต้องจดเอง ตรวจสอบยอดเงินด้วยมือ ให้เป็นระบบอัตโนมัติ เพื่อความรวดเร็วและแม่นยำ โดยแก้ปัญหาหลักดังนี้
* **งานซ้ำซ้อนและเสียเวลา:** ต้องคอยดาวน์โหลดรูปสลิปแล้วอัปโหลดขึ้น Drive เอง หรือต้องพิมพ์ข้อมูลจากภาพลง Excel/Sheets ด้วยมือ
* **ไม่ชัดเจน:** กรณีมียอดโอนเท่ากัน ในเวลาไล่เลี่ยกัน ยากที่จะระบุว่ายอดไหนเป็นของลูกค้าคนไหน
* **ความผิดพลาดจากคน (Human Error):** พิมพ์ตัวเลขผิด หรือจำแนกหมวดหมู่รูปภาพผิดพลาด
* **ข้อมูลตกหล่น / หาไม่เจอ:** ข้อมูลมักกระจัดกระจาย ค้นหาย้อนหลังยาก

## 🌟 ประโยชน์ของระบบ (System Benefits)
* **ลดงาน Manual:** เปลี่ยนงานซ้ำๆ ที่ต้องใช้คนทำ (ดาวน์โหลด-อัปโหลด-พิมพ์ข้อมูล) ให้กลายเป็นระบบอัตโนมัติ
* **ทำงานได้ 24/7:** ระบบทำงานได้ตลอดเวลาไม่มีวันหยุด ไม่ต้องพักเที่ยง
* **ลดความผิดพลาด:** ลดปัญหา Human Error จากการกรอกข้อมูลผิด หรือจำรายละเอียดในภาพพลาด เพราะใช้ AI เป็นตัวช่วยคัดกรอง
* **จัดเก็บเป็นระบบ:** ไฟล์ถูกเก็บเข้าที่ ข้อมูลถูกเขียนลงตารางอย่างเป็นระเบียบ ทำให้เรียกดูย้อนหลังหรือค้นหาได้ง่าย
* **พร้อมใช้งานทันที:** ข้อมูลสามารถนำไปทำสรุปสถิติ หรือวิเคราะห์แนวโน้มต่างๆ ได้เลยโดยไม่ต้องมานั่งคีย์ข้อมูลใหม่

## 🔄 ขั้นตอนการทำงานของระบบ (Workflow)
ระบบทำงานสอดประสานกันผ่าน n8n เวิร์กโฟลว์ โดยแบ่งออกเป็น 7 ขั้นตอนหลัก

<br>

*   **📱 01 | LINE Messaging Trigger**
    ระบบเริ่มทำงานทันทีแบบ Real-time เมื่อตรวจพบว่าลูกค้าส่งรูปภาพสลิปเข้ามาทาง LINE OA

*   **📥 02 | LINE Messaging Data Extraction**
    ดึงข้อมูลสำคัญจาก Webhook ของ LINE เช่น ข้อความ, Binary รูปภาพสลิป และ LINE User ID

*   **⚡ 03 | กระบวนการทำงานขนาน (Parallel Processing)**
    *   **`3A` Analyze an Image (AI Vision):** ส่งรูปภาพสลิปให้ AI ประมวลผลเพื่อดึงข้อมูลอัตโนมัติ อันได้แก่ *ชื่อผู้โอน, วันที่/เวลาโอน, และจำนวนเงิน*
    *   **`3B` Upload File (Google Drive):** อัปโหลดไฟล์รูปภาพไปจัดเก็บอย่างปลอดภัยบน Google Drive และส่งชุด URL ลิงก์รูปภาพกลับมาใช้งานต่อ

*   **🔗 04 | Merge Data**
    รวบรวมข้อมูลทั้งหมด (ข้อมูลสลิปจาก AI และ ลิงก์รูปภาพจาก Google Drive) มารวมเข้าด้วยกันเป็นชุดข้อมูล (Object) เดียว

*   **💻 05 | JavaScript Validation Engine**
    นำชุดข้อมูลมาผ่านโค้ด JavaScript เพื่อตรวจสอบความถูกต้องเชิงลึก เช่น **การตรวจสลิปซ้ำ (Duplicate Check)** เพื่อป้องกันการทุจริตหรือส่งข้อมูลซ้อน

*   **📊 06 | Append Row in Google Sheets**
    บันทึกข้อมูลที่ผ่านการตรวจสอบเรียบร้อยแล้วลง Google Sheets แยกคอลัมน์ชัดเจน เพื่อใช้เป็นฐานข้อมูลหลัก

*   **💬 07 | LINE Messaging (Send Reply)**
    ยิง API ส่งข้อความตอบกลับแจ้งสถานะการตรวจสอบให้ลูกค้าทราบผ่าน LINE ทันทีแบบอัตโนมัติ

## 🚀 วิธีติดตั้งและรันระบบ (Deployment & Setup Guide)

เนื่องจากระบบนี้เป็นรูปแบบ **Workflow Automation** ที่ขับเคลื่อนด้วย **n8n** เป็นหลัก การเปิดใช้งานระบบจึงจำเป็นต้องตั้งค่าความเชื่อมโยง (Integration) ระหว่างแพลตฟอร์มต่าง ๆ ตามขั้นตอนดังนี้

### 1. การเตรียมความพร้อมด้านสิทธิ์การเข้าถึง (Credentials & API Keys)
ก่อนเริ่มรันเวิร์กโฟลว์ ให้เปิดสิทธิ์และคัดลอกคีย์สำคัญจากบริการต่าง ๆ ดังนี้:
* **LINE Developers:** สมัครบัญชีและสร้าง Messaging API Channel จากนั้นคัดลอก **Channel Access Token** และ **Channel Secret**
* **Google Cloud Console:** เปิดใช้งาน Google Drive API และ Google Sheets API จากนั้นสร้างสิทธิ์การเข้าถึงแบบ **OAuth 2.0** หรือ **Service Account** เพื่อนำมาเชื่อมต่อกับ n8n
* **AI Vision Provider:** เตรียม API Key ของค่าย AI ที่คุณเลือกใช้งาน (เช่น OpenAI GPT-4o, Anthropic Claude, หรือบริการ OCR/Vision อื่น ๆ)

### 2. การนำเข้าเวิร์กโฟลว์สู่ n8n (Import Workflow)
1. เปิดโปรแกรมหรือหน้าเว็บ **n8n** (รองรับทั้งแบบ Cloud และ Self-hosted เช่น Docker)
2. คลิกสร้างเวิร์กโฟลว์ใหม่ (**Create a workflow**)
3. ไปที่เมนูมุมขวาบน เลือก **Import from File** (หากเซฟเวิร์กโฟลว์ไว้เป็นไฟล์ `.json`) หรือใช้การคัดลอกโค้ด JSON ของเวิร์กโฟลว์แล้วกด **Ctrl + V** (หรือ Cmd + V บน Mac) เพื่อวางลงบนหน้าต่าง n8n ได้ทันที

### 3. การเชื่อมต่อโหนดต่าง ๆ (Nodes Configuration)
กดเปิดเข้าไปที่แต่ละโหนดใน n8n เพื่อทำการผูกบัญชีผู้ใช้ (Add Credential):
* **โหนด LINE:** กรอก Access Token และ Channel Secret ที่เตรียมไว้
* **โหนด Google Drive / Sheets:** ทำการ Sign-in ผ่าน Google OAuth เพื่ออนุญาตให้ n8n มีสิทธิ์สร้างไฟล์และบันทึกข้อมูลลงตาราง Sheets
* **โหนด AI Vision:** เลือกโมเดลที่ต้องการใช้งาน พร้อมทั้งใส่ข้อความคำสั่ง (Prompt) เพื่อกำหนดให้ AI สกัดเฉพาะค่าที่ต้องการ เช่น ชื่อ, วันที่, และจำนวนเงิน กลับมาในรูปแบบ JSON Object

### 4. การตั้งค่า Webhook บน LINE OA
เพื่อให้ LINE สามารถส่งรูปภาพสลิปมาหา n8n ได้ทันที ต้องทำการเชื่อมลิงก์เข้าหากัน:
1. ดับเบิลคลิกที่โหนด **LINE Messaging Trigger** ใน n8n
2. คัดลอกลิงก์ **Webhook URL** (แนะนำให้ใช้ *Production URL* เพื่อการรันจริง หรือ *Webhook Test URL* หากอยู่ในขั้นตอนทดสอบ)
3. นำลิงก์ดังกล่าวไปวางในช่อง **Webhook URL** ที่หน้าตั้งค่า Messaging API บนเว็บไซต์ **LINE Developers Console**
4. กดปุ่ม **Verify** เพื่อทดสอบสัญญาณเชื่อมต่อ และตรวจสอบว่าได้เปิดสวิตช์ **Use Webhook** เป็นเปิด (Enabled) เรียบร้อยแล้ว

### 5. การทดสอบและเปิดใช้งานจริง (Testing & Activation)
* **ขั้นตอนทดสอบ (Testing):** คลิกปุ่ม **"Listen for test event"** บนโหนด LINE Messaging Trigger ใน n8n จากนั้นทดลองใช้บัญชี LINE ส่วนตัวส่งรูปภาพสลิปเข้าไปยัง LINE OA ของระบบ แล้วตรวจสอบในหน้า n8n ว่าข้อมูลไหลผ่านโหนดต่างๆ (01 จนถึง 07) ครบถ้วนและไม่มีจุดใดขึ้นสถานะ Error
* **ขั้นตอนเปิดใช้งานจริง (Production):** เมื่อทดสอบระบบจนมั่นใจว่าแม่นยำแล้ว ให้สับสวิตช์มุมขวาบนของหน้าจอ n8n จาก **Inactive** ให้เป็น **Active** > ⚡ **ระบบพร้อมทำงานอัตโนมัติ 100% ตลอด 24 ชั่วโมงเรียบร้อยแล้ว!** หลังจากนี้เมื่อมีลูกค้าส่งสลิปเข้ามา ระบบจะทำการตรวจสอบ บันทึก และส่งข้อความตอบกลับโดยที่คุณไม่ต้องเปิดหน้าจอ n8n ทิ้งไว้
