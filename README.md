# ระบบตรวจสอบสลิปโอนเงินอัตโนมัติผ่าน LINE OA (Automated Bank Slip Verification via LINE OA)

ระบบทำงานอัตโนมัติ 100% ตั้งแต่รับสลิป จนบันทึกข้อมูล และตอบกลับลูกค้าแบบทันทีผ่าน LINE Official Account โดยใช้เทคโนโลยี n8n ร่วมกับ AI Vision

---

## 📌 แนวคิดและปัญหาที่พบ (Concept & Pain Points)
เปลี่ยนการเก็บข้อมูลสลิปแบบเดิมๆ ที่ต้องจดเอง ตรวจสอบยอดเงินด้วยมือ ให้เป็นระบบอัตโนมัติ เพื่อความรวดเร็วและแม่นยำ โดยแก้ปัญหาหลักดังนี้:
* **งานซ้ำซ้อนและเสียเวลา:** ต้องคอยดาวน์โหลดรูปสลิปแล้วอัปโหลดขึ้น Drive เอง หรือต้องพิมพ์ข้อมูลจากภาพลง Excel/Sheets ด้วยมือ
* **ไม่ชัดเจน:** กรณีมียอดโอนเท่ากัน ในเวลาไล่เลี่ยกัน ยากที่จะระบุว่ายอดไหนเป็นของลูกค้าคนไหน
* **ความผิดพลาดจากคน (Human Error):** พิมพ์ตัวเลขผิด หรือจำแนกหมวดหมู่รูปภาพผิดพลาด
* **ข้อมูลตกหล่น / หาไม่เจอ:** ข้อมูลมักกระจัดกระจาย ค้นหาย้อนหลังยาก

---

## 🌟 ประโยชน์ของระบบ (System Benefits)
* **ลดงาน Manual:** เปลี่ยนงานซ้ำๆ ที่ต้องใช้คนทำ (ดาวน์โหลด-อัปโหลด-พิมพ์ข้อมูล) ให้กลายเป็นระบบอัตโนมัติ
* **ทำงานได้ 24/7:** ระบบทำงานได้ตลอดเวลาไม่มีวันหยุด ไม่ต้องพักเที่ยง
* **ลดความผิดพลาด:** ลดปัญหา Human Error จากการกรอกข้อมูลผิด หรือจำรายละเอียดในภาพพลาด เพราะใช้ AI เป็นตัวช่วยคัดกรอง
* **จัดเก็บเป็นระบบ:** ไฟล์ถูกเก็บเข้าที่ ข้อมูลถูกเขียนลงตารางอย่างเป็นระเบียบ ทำให้เรียกดูย้อนหลังหรือค้นหาได้ง่าย
* **พร้อมใช้งานทันที:** ข้อมูลสามารถนำไปทำสรุปสถิติ หรือวิเคราะห์แนวโน้มต่างๆ ได้เลยโดยไม่ต้องมานั่งคีย์ข้อมูลใหม่

## 🔄 ขั้นตอนการทำงานของระบบ (Workflow)
ระบบทำงานสอดประสานกันผ่าน n8n เวิร์กโฟลว์ โดยแบ่งออกเป็น 7 ขั้นตอนหลัก:

1. **Line Messaging Trigger:** ระบบเริ่มทำงานทันทีเมื่อลูกค้าส่งรูปภาพสลิปการโอนเงินเข้ามาทาง LINE OA
2. **Line Messaging Data:** ดึงข้อมูลและไฟล์รูปภาพจากข้อความใน LINE (ข้อความ, รูปภาพสลิป, User ID)
3. **กระบวนการทำงานขนาน (Parallel Processing):**
   * **3A. Analyze an image (AI Vision):** AI วิเคราะห์และอ่านข้อมูลจากสลิปอัตโนมัติ ดึงค่า *ชื่อผู้โอน, วันที่โอน, และจำนวนเงิน*
   * **3B. Upload file (Google Drive):** อัปโหลดไฟล์รูปภาพสลิปไปเก็บที่ Google Drive เพื่อเป็นหลักฐานรอง และส่ง URL ลิงก์รูปภาพกลับมา
4. **Merge:** รวบรวมข้อมูลจาก AI และลิงก์รูปภาพเข้าด้วยกัน
5. **Code in JavaScript:** ตรวจสอบเงื่อนไขและความถูกต้องของข้อมูล เช่น ตรวจสอบรายการซ้ำ (Duplicate Check)
6. **Append row in sheet:** บันทึกข้อมูลที่ผ่านการตรวจสอบลง Google Sheets (ชื่อผู้โอน, วันที่โอน, จำนวนเงิน, หลักฐานการโอนจาก Google Drive)
7. **Line Messaging (Send):** ส่งข้อความตอบกลับหาลูกค้าผ่าน LINE ทันที

---

## 📊 ผลลัพธ์ที่ได้ (Final Output)
* **ระบบ LINE OA:** ส่งข้อความยืนยันการบันทึกสลิปเรียบร้อย พร้อมสรุปรายละเอียดวันที่โอน ชื่อผู้โอน และจำนวนเงินให้ลูกค้าทราบทันที
* **Google Sheets:** เก็บข้อมูลตารางสรุป รายวัน รายชื่อ ยอดเงิน พร้อมแปะลิงก์รูปภาพสลิปหลักฐานอย่างชัดเจน
* **Google Drive:** จัดเก็บรูปภาพสลิปแยกเป็นรายไฟล์อย่างเป็นระบบบนคลาวด์ พร้อมระบุวันเวลาที่อัปโหลด
